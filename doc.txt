Executive Summary
This project presents RevXcel, a smart marketing intelligence system designed to address the low
conversion rates in e-commerce using big data analysis. Inspired by a real case where only 2% of
website visitors completed purchases, the system leverages big data technologies, machine
learning models, and real-time dashboards to provide actionable insights. By integrating data
from social media, website analytics, and email campaigns through APIs, RevXcel enables
marketers to predict customer behavior, evaluate campaign performance, and improve ROI.
Developed using Python-based tools and SAS Visual Analytics, the project follows the Design
Science Research methodology and adopts the Waterfall model for structured development. The
outcome is a scalable and interactive platform that supports data-driven decision-making in
digital marketing.

1. Introduction
The digital world is witnessing a significant increase in website traffic; however, the conversion
rate to actual buyers remains low. This is evident in the case of Abdulrahman Al-Olayan, where
the company received over 50,000 visitors but achieved only a 2% purchase rate. Based on this
challenge, the RevXcel system was developed to address the gap in real-time behavioral data
analysis. It integrates big data technologies and machine learning to support marketing and sales
decisions and improve return on investment (ROI).
This report started with a project background, followed by a review of similar projects in the
same area. After that, the gap has been identified in alignment with the aim, objectives, and
benefits of the project. The project methodologies have been introduced using the Design
Science Methodology (SDR) and the Waterfall framework. Then, the system was designed using
UML and UI. Finally the Gantt chart presented.
2. Project Overview, Background, and Motivation
2.1 Project Overview
Despite increasing digital traffic (as seen in Abdulrahman Al-Olayan's case with over 50k site
visitors), only a small fraction (2%) convert to buyers. Businesses lack a unified system to analyze
behavioral data in real time to drive smarter marketing decisions. RevXcel is designed to solve
this gap using integrated big data and machine learning to significantly improve conversion and
ROI.
2.2 Background
With the significant deployment of digital transformation, marketing and sales departments in
large organizations are seeking smarter ways to reach customers. The availability of massive
customer data from websites and social platforms has created opportunities to make data-driven
decisions. However, organizations still struggle to unify and analyze these diverse datasets
effectively. Recent developments in machine learning, API-based integrations, and big data
platforms like Spark and Hadoop provide the tools necessary to bridge this gap. If not addressed,
companies may continue making decisions based on fragmented or outdated data, leading to
lower ROI and customer dissatisfaction.
2.3 Motivation
This project was inspired by a real-world case from Abdulrahman Al-Olayan, who faced a critical
issue: despite having over 50,000 visitors to the company's online store (Nice One), only 2%
converted to actual buyers. This highlighted a major problem in customer targeting and
engagement. RevXcel is designed to fix that — by leveraging big data. We aim to significantly
boost purchase rates, personalize the customer journey, and drive better outcomes for both the
company and its customers.

3.1 Project 1: GLARE - Google Play Reviews in Arabic
Description and Use case:
GLARE is a comprehensive database of 76 million Arabic reviews from Google Play. It analyzes
user behavior to extract insights about app performance, trends, and user satisfaction. The
primary use is to support marketing and development decisions.
Data Sources & Volume:
The data used in this project are obtained via the Google Play Store for 76 million Arabic reviews.
Source: Google Play Store
Data Volume: 76 million Arabic user reviews
Languages: Arabic dialects and Modern Standard Arabic (MSA)
Technologies used :
As for any machine learning projects, the following techniques are mandatory to apply them and
work on any ML project.
Natural Language Processing (NLP)
Python
Classification & Clustering Algorithms
Big Data Management Tools
Apache Spark (for scalable data processing)
Architecture and Performance matrices:
1. Data Collection Layer
In the data collection section, the team collected the data using custom web scraping and
Google Play APIs for the Arabic reviews precisely.
2. Preprocessing Layer
The preprocessing tasks focused mainly on the following:
Text normalization and cleaning
Removal of ads, emojis, icons, and misspellings
Tokenization and duplicate review removal
3. NLP Feature Engineering
For the sake of feature engineering the team mainly focused on the following things, the
sentiment analysis (polarity and emotion detection), keyword extraction, and Context modeling
using Arabic NLP libraries (e.g., Farasa, CAMeL Tools)
4. Classification & Grouping
The classification process focused only on two main things which are sentiment detection
(positive/negative) and topic classification (e.g., features, bugs, usability, performance)
5. Analytics & Reporting
Finally, the reporting included an interactive dashboard, an actionable insight for marketing and
teams, real-time summaries and feedback on clustering. Additionally, a user trend analysis was
provided.

9. Project Feasibility

Our project is technically feasible due to the availability of mature tools, libraries, and open-
source technologies. We utilized Python-based machine learning frameworks such as Scikit-learn,

Pandas, SQLite, or scalable options like Google BigQuery if needed. All required components are

9.2 Operational Feasibility
From an operational perspective, our solution fits naturally into the marketing and data
workflows of modern businesses. The dashboard provides intuitive access to customer insights,
which supports marketers in making timely decisions. The use of predictive models enhances
campaign planning without requiring deep technical knowledge from end users. Our design
ensures the system can be maintained with moderate technical support, and future iterations
can be scaled based on business needs.
9.3 Economic/Financial Feasibility
We focused on using free and open-source technologies to minimize cost. Libraries such as Scikit-
learn, Dash, and Pandas eliminated licensing fees. The project was developed on personal
computing devices, and we used sample or publicly available data for testing. If deployed in a
21
real business context, additional costs may include cloud hosting, paid APIs, or advanced analytics
services — but for this phase, the financial risk remains low and manageable.
9.4 Legal & Ethical Feasibility
The project complies with basic data privacy and ethical standards. While we used simulated or
anonymized data, in real-world applications, it is critical to comply with data protection laws such
as GDPR (General Data Protection Regulation) or local data privacy regulations. Our system
assumes that organizations have legal consent to collect and process customer data. Ethically,
our project promotes transparency and responsible use of personal information to enhance
customer experience, not exploit it.

11. Project Methodology

a structured yet flexible approach that is ideal for developing innovative technical solutions to
real-world problems. DSR focuses on building and evaluating artifacts that solve identified
challenges, which perfectly align with the goal of our RevXcel system (Anderson M , 2015 ).

We began by identifying a real-world challenge: despite high online traffic, companies
like Nice One experience very low conversion rates due to poor customer targeting
and fragmented marketing insights.
2. Define Objectives for a Solution
Our objective was to build a system that integrates diverse customer data sources and
23
Apply machine learning to generate real-time marketing insights, improving both

3. Design and Development
We designed and built the RevXcel system to unify customer data, apply predictive

4. Demonstration
We demonstrated our solution using both real and simulated data, showing how it can be

The results of our work are communicated through dynamic dashboards (e.g., Plotly

Stakeholders.

phase must be completed before moving to the next. It is best suited for projects with clear
objectives, stable requirements, and a predictable timeline. (Royce,1970).
I. Gathering and Analysis Requirements:
Functional and non-functional requirements were collected from stakeholders, including the
marketing team, data scientists, and IT personnel. In the RevXcel project, this phase focused on
identifying pain points such as low conversion rates despite high online traffic.
II. System Design:
The gathered requirements were translated into a system architecture. This included designing
the data flow between social media platforms, website analytics, and dashboards.
III. Implementation (Development):
The system was developed using selected technologies such as Python, Pandas, Scikit-learn,
XGBoost, and Dash for building machine learning pipelines and interactive dashboards
IV. Testing:
Testing involved evaluating prediction accuracy, dashboard interactivity, and the reliability of
data integration using both unit and system-level tests.
V. Deployment:
The developed platform was demonstrated using both real and simulated datasets to showcase
its applicability in a marketing context.
VI. Maintenance:
While not a major part of the project scope, the solution was designed with modular and
maintainable code to support future updates and scalability.

14. Functional Requirements:
14.1 From Administrator's Perspective
The administrator in the system has key responsibilities, including approving or rejecting vendor
applications and verifying distinguished vendors to enhance visibility. They also have the
authority to delete vendor accounts based on reviews or policy violations, ensuring a safe and
trustworthy platform.
14.2 From Vendor Perspective
Vendors play a crucial role in providing services to customers through the platform. To ensure a
seamless experience, several functions are integrated. These include account creation, service
offering, and product management, which are essential for vendors to maintain their presence
and offerings.
14.3 From Clients Perspective
Clients serve as the primary users of the e-platform, gaining access to many services offered by
vendors. To enhance their experience, the system allows clients to browse and view products
(services) and vendor details without the need to log in initially. This ensures ease of navigation
and smooth interaction with the services.
14.4 Non-Functional Requirements
• Scalability: Ability to scale horizontally for increased user and vendor numbers.
• Security: Encrypt user data and implement secure authentication protocols.
• Usability: Intuitive interface for easy navigation with minimal training.
• Maintainability: Modular and well-documented codebase for easy updates.
• Compatibility: Functionality across major web browsers and mobile platforms.
• Localization: Support for Arabic and English languages initially.

Explain
1. Login Process
 The MarketingUser sends a login request to the system.
 The system forwards this to the Auth Module for validation.
 If successful, the system responds with access to the dashboard.
 This step ensures that only authorized users access the data.
2. Analyze Campaign Performance
 The user initiates an analysis request (e.g., to view how ads are performing).
 The system delegates this to the Analytics Engine.
 The analytics engine retrieves campaign data, processes it, and returns insights (e.g.,
click-through rates, cost per lead).
 The user receives visual metrics and analysis results.
3. Predict Customer Behavior
 The MarketingUser wants to understand future behavior (e.g., segment prediction,
churn likelihood).
 The system calls the ML Model, which runs trained algorithms (like XGBoost or Random
Forest).
 The output includes predicted customer segments or scores (e.g., High-Value, At-Risk).
 This prediction is sent back to the user for interpretation.

17. User Scenario
Scenario: Marketing User Journey – From Login to Insight
A marketing analyst logs into the RevXcel platform using their company email and password.
After successful authentication, the system redirects the user to the main dashboard, where a
high-level view of current performance metrics is displayed, including:
 ROI (Return on Investment)
 CLV (Customer Lifetime Value)
 Engagement Rate
 Top-performing Channels
The user navigates to the Campaign Analysis section via the sidebar. Here, they can:
 Select from a list of active and archived campaigns
 View key statistics such as click-through rate, conversion rate, and budget usage
 Visualize ROI across multiple marketing channels (email, social media, paid ads)
To explore customer behavior further, the user clicks “Predict Segments”, which runs the ML
model in the background. Within seconds, RevXcel returns segmented predictions such as:
 High-potential leads
 At-risk customers
 Frequent buyers
Each segment includes detailed descriptions, graphs, and a suggested marketing strategy.
From the Reports tab, the user clicks "Generate Report". The system compiles the selected
campaign data, ML outputs, and dashboard visuals into a downloadable PDF report.
Finally, the marketing user returns to the dashboard, where updated insights are now visible.
They use these results to schedule a team meeting and adjust ad spending based on the
predicted customer behavior.

20. Conclusion
The RevXcel project demonstrates how big data and machine learning technologies can address
the issue of low conversion rates in digital marketing. By integrating data from social media and
website analytics, and presenting predictive, interactive dashboards, the system delivers
actionable insights that help marketing and sales teams enhance performance and increase ROI.
This report started with a project background, followed by a review of similar projects in the
same area. After that, the gap has been identified in alignment with the aim, objectives, and
benefits of the project. The project methodologies have been introduced using the Design
Science Methodology (SDR) and the Waterfall framework. Then, the system was designed using
UML and UI. Finally the Gantt chart presented.
The use of open-source tools and API-based integration also supports the feasibility of
implementing this solution in real business environments.
